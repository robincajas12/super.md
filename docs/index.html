SUPER.MD

A virtual filesystem that attaches computable metadata and execution layers to Markdown files.

super.md Node mounts a directory via FUSE and intercepts file read operations. Content is processed on-demand by executing embedded code blocks.

Usage

Prerequisites

  • Node.js >= 14
  • FUSE

Start

node index.js <project_directory>

If it is the first time you open this project, it will ask for the path where you want your virtual files to live.

All files must use the .super.md extension; standard .md files will behave normally and remain unaffected.

In fact, every other file besides .super.md should work perfectly as normal files.

You can use multiple code blocks in a single file.

State Management

  • Edit Mode (1/e): Standard source text.
  • Exec Mode (2/x): Process output.

All .super.md files can be edited in Edit Mode but are read-only in Exec Mode.

Execution Blocks

run-node

# Title
```run-node
console.log("## 1 + 1: ", 1 + 1);
```

Running cat file_name.super.md should return this:

# Title
## 1 + 1: 2

run

```run
echo "**hello**"
```

Running cat file_name.super.md should return this:

**hello**

script

```script
./my-script
```

Caching

Append a duration to a block tag. Syntax: :time (e.g., :1h).

Time units supported:

  • seconds: s
  • minutes: m
  • hours: h
  • days: d

This block stores its output for 10 minutes. If 10 minutes have passed when reading the file, the code will execute again.

```run:10m
echo "cached result"
```
